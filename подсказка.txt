
1) создать папку с новым проектом

2) скорипровать/вставить шаблон, либо клонировать с гитхаба - git clone <URL>

3) проверить наличие:

папки .vscode - это допнастройки как проводить импорты, настройки шрифтов, хлебные крошки и тд.
файла .pylintrc - это файл настроек для пайлинтера (подсказки при написании кода, pip8, список не нужных напоминалок/ошибок)
файла prompt_config.sh - это настройки для вывода красивого промта в терминале
.gitignore - список того что не нужно учитывать в системе контроля версий git
LICENSE - стандартная лицензия (хз зачем, пока просто так)

4) создать папку venv с виртуальным окружением через терминал

python -m venv venv
python3.11 -m venv venv

5) запустить виртуальное окружение

source venv/Scripts/activate  - для GitBash
venv\Scripts\activate         - для Windows

deactivate                    - выйти из вируального окружения

6) проверить актуальные версии

python --version
pip --version
git --version
docker --version


7) скачать и установить расширения requirements

pip freeze > requirements.txt         - создать файл requirements.txt из ВСЕХ зависимостей в env

pip install -r requirements.txt       - скачать все расширения из файла

pip install pipreqs                   - скачать pipreqs
pipreqs . --force                     - создать файл requirements.txt из ИСПОЛЬЗУЕМЫХ зависимостей проекта

pip uninstall <github>                - удалить определенную зависимость из env


8) система контроля версий Git

git init
git pull

git add .
git commit -m "text commit"

git push                       — отправляет локальные коммиты в удаленный репозиторий
git rebase                     — синхронизировать локальную ветку с удаленной (используется для изменения локальных коммитов)
git fetch                      — обновления информации о состоянии удаленного репозитория

rm -rf .git                    — отключить систему контроля версий Git


9) создание исполняемого файла

pip install pyinstaller            - качаем необходимую библиотеку
version_info.txt                   - создаем текстовый файл с данными об авторстве

pyinstaller --onefile --windowed --add-data "images/*;images" --version-file version_info.txt main.py
